# Invoice Billing Period + Company Name — Design

**Date:** 2026-06-02
**Status:** Approved (pending spec review)
**Project:** freelance-tracker

## Problem

Invoices generated by the app do not show a **billing period**. The period is only inferable indirectly — from the free-text `notes` line and from date-prefixed line-item descriptions — but there is no explicit, structured "Billing Period" on the rendered PDF. Clients (e.g. SiFive) expect a clear statement of the period an invoice covers.

Separately, the invoice's **FROM** block can show a logo and a personal name, but there is **no dedicated company-name field**. A freelancer billing under a business name (e.g. "Inspired Ideation Strategies") currently has to jam it into the personal `name` field.

## Goals

1. Add an explicit **Billing Period** to invoices, set as **calendar-month bounds** (e.g. April 1 – April 30, 2026), captured at creation, editable afterward, and rendered on the PDF.
2. Add a dedicated **company name** to the freelancer profile, rendered prominently in the invoice FROM block.
3. Backfill the two existing SiFive invoices (INV-2026-001 = April, INV-2026-002 = May) with their periods.

## Non-Goals

- No change to how line items, totals, tax, or due dates work.
- No multi-month or arbitrary partial-range periods (calendar-month semantics only).
- No change to the client-side BILL TO block (clients already have a `company` field).
- No change to the logo upload/storage flow.

## Decisions (from brainstorming)

- **Period semantics:** calendar month — `period_start` = first day of month, `period_end` = last day of month.
- **Builder UX:** a single month picker (`<input type="month">`), defaulting to the month of the latest selected time entry. Chosen over dual date inputs (invites inconsistent ranges) and fully-automatic derivation (breaks for multi-month / expense-only / monthly-retainer invoices).
- **Storage:** on the `invoices` record (two nullable DATE columns) so the PDF renders reliably without re-querying time entries.
- **Company name:** its own profile field; rendered as the bold primary line of FROM with personal name beneath.

## Design

### 1. Schema

New migration file `freelance-tracker/supabase_migration_invoice_period.sql`:

```sql
ALTER TABLE invoices
  ADD COLUMN IF NOT EXISTS period_start DATE,
  ADD COLUMN IF NOT EXISTS period_end   DATE;
```

Both nullable. Existing invoices keep `NULL` and simply render no billing-period line. Applied to the live Supabase project (`pnilvktjzpnyqhnowuhs`) via the MCP `apply_migration`.

### 2. Types — `src/hooks/useInvoices.ts`

- `Invoice` interface: add `period_start: string | null` and `period_end: string | null`.
- `InvoiceInsert` (an `Omit` of `Invoice`) picks these up automatically.
- `InvoiceUpdate` pick: add `period_start` and `period_end` so the edit dialog can persist them.
- `createInvoice` already spreads the insert object, so no logic change is needed beyond the type.

The `useInvoices` / `useInvoice` queries use `select('*', ...)`, so the new columns propagate to the PDF caller without query changes.

### 3. Invoice Builder — `src/components/InvoiceBuilder.tsx`

- Add state `billingMonth: string` (format `YYYY-MM`).
- Default: the month of the most recent selected time entry; if no entries (monthly retainer / expense-only), default to the current month. Recompute the default when selections load, but do not override a value the user has manually changed.
- Render an `<input type="month">` labeled **"Billing month"** in the settings grid, next to Due Date.
- On `handleGenerate`, derive:
  - `period_start` = first day of `billingMonth` (`YYYY-MM-01`).
  - `period_end` = last day of that month (compute via `new Date(year, month, 0)`).
- Pass `period_start` / `period_end` in the `createInvoice` insert payload (both hourly and monthly branches).

### 4. Edit dialog — `src/components/InvoiceEditDialog.tsx`

- Add `billingMonth` state, hydrated from `invoice.period_start` (→ `YYYY-MM`).
- Render the same **"Billing month"** month-picker.
- On save, convert back to first/last-of-month and include `period_start` / `period_end` in the `updateInvoice` call. Empty picker → both `null`.

### 5. Company name — `src/pages/Settings.tsx` + `src/components/InvoicePDF.tsx`

- `Settings.tsx`: extend the `FreelancerProfile` interface with `company: string`; default `''`; add a **"Company name"** input above the existing Name field; include it in the `freelancer_profile` save payload.
- `InvoicePDF.tsx`: read `profile.company`. In the FROM block:
  - If `company` is set: render `company` in bold (size 11) as the first line, then `profile.name` in normal weight (size 9) on the next line, then email / phone / address as today (shifted down one line).
  - If `company` is empty: keep current behavior (name in bold first).

### 6. PDF billing period — `src/components/InvoicePDF.tsx`

- After the **Project** line (`yPos = 88`), add a billing-period line at `yPos = 92` (nudge the line-items table `startY` accordingly if needed to avoid collision).
- Render only when both `invoice.period_start` and `invoice.period_end` are present:
  `Billing Period: April 1 – April 30, 2026`
- Use a small range formatter that avoids repeating the year when start and end share it.
- New i18n key `invPdf.period` (value `'Billing Period'` / `'Período de facturación'`) added to both the `en` and `es` dictionaries in `src/lib/i18n.tsx`.

### 7. Backfill (live data)

```sql
UPDATE invoices SET period_start = '2026-04-01', period_end = '2026-04-30'
  WHERE invoice_number = 'INV-2026-001';
UPDATE invoices SET period_start = '2026-05-01', period_end = '2026-05-31'
  WHERE invoice_number = 'INV-2026-002';
```

## Components & Responsibilities

| Unit | Responsibility | Depends on |
|------|----------------|------------|
| `supabase_migration_invoice_period.sql` | Adds period columns | invoices table |
| `useInvoices.ts` (types) | Carries period fields through insert/update | Supabase row shape |
| `InvoiceBuilder.tsx` | Captures billing month → period bounds at creation | selected entries, `createInvoice` |
| `InvoiceEditDialog.tsx` | Edits period post-creation | `updateInvoice` |
| `Settings.tsx` | Captures company name into profile | `userStorage` |
| `InvoicePDF.tsx` | Renders FROM (company+name) and Billing Period line | profile, invoice fields, i18n |
| `i18n.tsx` | `invPdf.period` label (en/es) | — |

## Edge Cases

- **Old invoices / no period:** both columns null → no billing-period line renders. No error.
- **Multi-month selections:** builder defaults to the latest entry's month; user adjusts the single picker if they want a different month. Period is always a clean calendar month.
- **Monthly-retainer or expense-only invoices:** no time entries → default to current month; user can change.
- **Empty company name:** PDF falls back to current name-first layout.
- **Locale:** the period range and month label format via the existing locale (`en-US` / `es-ES`).
